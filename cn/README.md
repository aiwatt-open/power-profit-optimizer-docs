# API 接口文档

电力利润优化系统（Power Profit Optimizer）的中文 API 接口文档。用于对接站点管理、设备管理、功率数据及最大利润优化等能力。

## 文档索引

| 文件 | 模块 | 说明 |
|------|------|------|
| [site.md](site.md) | 站点管理 | 站点的增删改查 |
| [device.md](device.md) | 设备管理 | 设备的增删改查 |
| [power.md](power.md) | 电力数据 | 功率记录、最大利润优化 |

---

## 接口地址

文档中的路径为相对路径（如 `/v1/devices/`），实际请求需拼接服务 Base URL，例如：

```
https://your-domain.com/api/v1/devices/
```

即文档中的 `/v1/xxx` 对应实际 URL 的 `/api/v1/xxx`。

---

## 请求与响应

- **请求**：GET 使用 Query 参数；POST/PUT 使用 JSON Body；路径中的 `:id` 等为路径参数，需替换为实际值。
- **响应**：成功时返回 JSON；失败时同样为 JSON，格式见下方「错误响应」。

---

## 分页说明

列表类接口（如获取设备列表、功率记录列表）采用分页返回，统一字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| items | array | 当前页数据列表 |
| total | int | 总记录数 |
| page | int | 当前页码（从 1 开始） |
| page_size | int | 每页记录数 |
| total_pages | int | 总页数 |

请求时使用 `page` 和 `page_size` 控制分页，例如：`?page=1&page_size=10`。

---

## 时间字段说明

| 字段 | 格式 | 示例 |
|------|------|------|
| timestamp、start_time、end_time 等 | int，Unix 时间戳（秒） | `1736143200` |
| created_at、updated_at | string，ISO 8601 日期时间 | `"2025-01-06T10:00:00"` |

时间相关查询参数均使用 Unix 时间戳（秒）。

---

## 错误响应

请求失败时返回 JSON：

```json
{
  "code": 20001,
  "message": "Device not found"
}
```

| 字段 | 说明 |
|------|------|
| code | 错误码，用于区分具体错误类型 |
| message | 错误信息（英文） |

**错误码区间**：

| 区间 | 含义 |
|------|------|
| 10000-19999 | 参数或请求格式错误 |
| 20000-29999 | 资源不存在（如设备、站点） |
| 30000-39999 | 业务逻辑错误（如数据不足、类型不匹配） |
| 50000-59999 | 服务器内部错误 |

各接口文档的「错误码」表格中列出了该接口可能返回的错误码及其中文说明。
